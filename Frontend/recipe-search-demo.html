Overview

The recipe search functionality has been fully implemented in the Profile.html page with the following features:

Keyword Search

Search recipes by title, description, or any text content. Includes intelligent filtering by categories, meal types, and tags.

Ingredient-Based Search

Get recipe suggestions based on available ingredients. The system will find recipes you can make with what you have.

Advanced Filtering

Filter recipes by categories, meal types (breakfast, lunch, dinner), and tags for precise results.

Responsive Design

Fully responsive interface that works perfectly on desktop, tablet, and mobile devices.

API Integration

The frontend is configured to work with the following Django backend endpoints:

GET /api/recipes/?search=query&categories=cat&tags=tag&meal_type=type
POST /api/recipes/suggest-by-ingredients/
GET /api/recipes/categories/
GET /api/recipes/tags/

Backend Requirements

  • Django backend running on http://localhost:8000
  • CORS properly configured to allow frontend requests
  • Recipe API endpoints implemented and working
  • Authentication system (optional but recommended)

How to Test

1. Open Profile Page

Navigate to Profile.html and click on the "Find Recipes" tab

Open Profile

2. Try Keyword Search

Enter a recipe name or ingredient in the search box and apply filters

Example: "chicken curry" + Category: "Main Course"

3. Test Ingredient Search

Switch to "By Ingredients" tab and enter available ingredients

Example: "tomato, onion, chicken, rice"

4. Check Console

Open browser developer tools to see API calls and any error messages

F12 → Console → Network tabs
